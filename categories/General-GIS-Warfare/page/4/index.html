Numpy npz file stuff

For some raster analysis I’ve been storing datasets as numpy ndarrays in an npz file. Kind of like pickled but more numpy-ish. For some of the rasters I loaded, the null value came out to be 255 - I needed to change them to 0 so here’s what I came up with.

from numpy import *
from PIL import Image

#use this script to retro replace 255s with 0s in the input rasters.
f = load('C:/Scratch/inputs.npz')

replace={255:0}

bike = copy(f['bike'])
for k, v in replace.iteritems(): bike[f['bike']==k]=v

block = copy(f['block'])
for k, v in replace.iteritems(): block[f['block']==k]=v

savez('C:/Scratch/inputs2.npz',bike=bike, block=block)

Field Calculator @Area++

This one comes courtesy my co-worker Clint Chiavarini

If you use the Calculate Geometry function in an ArcMap table to calculate area, acres, etc. for shapefiles, here is another way to do it using the field calculator:

  1. At the top of the Field Calculator box, Click “Python” instead of VB Script
  2. Type (without quotes): “!shape.area!” This will give you international square feet (the same as the projection you’re working in).
  3. Or type: !shape.area@acres! to go directly to acres.
  4. The same works for lengths i.e. “!shape.length@kilometers!” or whatnot
    There is no “Calculate Geometry” tool in arctoolbox so if you’re creating a model, you need to use this method in the “Calculate Field” tool.

ArcMap Field Calculator RegEx

Been waiting a long time on this one. Not sure why it hadn’t occurred to me earlier

Snippet:

import re

def f(str):
    p=re.compile("(\d+-?\d+).+")
    m=p.match(str)
    try:
        return m.group(1)
    except Exception, e:
        return ""
