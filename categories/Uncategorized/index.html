A snippet and a thorn

So I’m trying to run a very simple script in arcpy that adds 4 bounding box (Xmin, Ymin, etc) fields and calculates them using the python !shape! object.
This, of course, works fine via the attribute table in ArcMap but fails in scriptland here:

arcpy.management.CalculateField(fc, "XMIN1", "!shape!.extent.xmin", "PYTHON")

This forum post informed me that I need to change the PYTHON part to PYTHON_9.3 to make this work.

import arcpy

fc = "c:/users/sainsbury/desktop/osmmonitor/OSMNet_10_by10W.shp"
arcpy.management.AddField(fc, "XMIN1", "DOUBLE")
arcpy.management.AddField(fc, "YMIN1", "DOUBLE")
arcpy.management.AddField(fc, "XMAX1", "DOUBLE")
arcpy.management.AddField(fc, "YMAX1", "DOUBLE")

arcpy.management.CalculateField(fc, "XMIN1", "!shape!.extent.xmin", "PYTHON_9.3")
arcpy.management.CalculateField(fc, "YMIN1", "!shape!.extent.ymin", "PYTHON_9.3")
arcpy.management.CalculateField(fc, "XMAX1", "!shape!.extent.xmax", "PYTHON_9.3")
arcpy.management.CalculateField(fc, "YMAX1", "!shape!.extent.ymax", "PYTHON_9.3")

Batch geocoding

Recently I was asked by an acquaintance of a friend what the best way to geocode millions of addresses would be. The friend was trying to put together a bid for a company who wanted to know how much it would cost and how long it would take.

I spent a while formulating my thoughts around this and here is what I came up with:

Option A

Pay ESRI $5000 for the use of their geocoding engine ($50*25,000) + whatever their commercial software licensing fee is(~$2000?) for using ArcGIS Desktop (you can’t use a student/eval copy for commercial use right?)
• Feel bad about paying ESRI an exorbitant amount of monay.
• Feel good about the quality of the results.
• Feel good about the legality/ethics of throwing 2.5 million records at their server.

Option B

Pay Yahoo or Google or some other established company offering a geocoding web service whatever their commercial licensing fee is (I have no idea but likely less than ESRI’s) and roll your own custom client to consume their geocoder.
• Feel good about not paying ESRI an exorbitant amount of monay.
• Feel good about the quality of the results.
• Feel good about the legality/ethics of throwing 2.5 million records at their server.

Option C

Find a free geocoding engine (Oregon State’s, geocode.us) and roll your own custom client to consume their geocoder.
• Feel good about not paying anyone an exorbitant amount of monay.
• Feel unsure about the legality/ethics of throwing 2.5 million records at it.
• Feel unsure about the quality of the results and perform some modest QA/QC.

SPROCS: to use or not to use

I’ve been attempting to learn PL/SQL lately in order to create some stored procedures in our Oracle 11g database. I discovered that the Oracle Data Provider (ODP.NET) library provided a substantial performance boost over the traditional ODBC library. The jury is still out over dynamic mid-tier SQL versus parameterized stored procs. This post is going to be a living list of the/my pros and cons associated with stored procedures and a .NET/Oracle platform specifically.

Pros:

  • Stored procedures feel right from a traditional perspective (eg. I’m used to them).
  • Stored procedures theoretically protect against SQL injection - though I’m not certain on the nitty gritty of how exactly.
  • Stored procedures are pre-compiled so you know your SQL is tight.
  • New developments to the OPD.NET library allow you to pass a .NET class as a user-defined type to a stored proc as an input parameter (that sounds weird, but I’m not sure how else to phrase it. maybe that’s a clue.) The class has to implement several interfaces(IOracleCustomType, INullable) and it and it’s members must be decorated with the OracleCustomTypeMapping attribute. I haven’t tested this but this codeproject stub lays it down pretty well.
  • I’ve always been taught that stored procedures are highly performant

  • I’m not doing Test-Driven Development
    Cons:

  • What the %$#@ is up with PL/SQL? Some pretty silly conventions. No continue in a try catch?

  • What the %$#@ is up with newer versions of SQL Developer not compiling stored procs that older versions can?
  • How married is my agency to Oracle? Is it better to decouple amidst platform volatility? (probably)
  • How convertible to T-SQL is PL/SQL?
  • Jeff Atwood’s blog post from 2004 (7 years ago, dear god am I really that far behind in this technology?) Is pretty edutaining, however, comment number 1 is a pretty good response.
  • I’m not doing Test-Driven Development
