WKT v WKID :: AGS


FeatureLayer fl = new FeatureLayer();
fl.Url = “http://navigator.state.or.us/ArcGIS/rest/services/Framework/Admin_Bounds/MapServer/3“;
int[] foo = {3};
fl.ID = “tempFeatures”;
fl.Visible = true;
fl.InitializationFailed += new EventHandler<EventArgs>(fl_InitializationFailed);
map.Layers.Add(fl);

with:











==Fail.


Why?




Querying features with a WKT as the outSR fails at 9.31 - apparently it’s fixed at 10…


but the large part of the problem revolves around ESRI’s parsing of the projected coordinate system of the data frame when publishing services.


The Oregon Metro Projection is NAD_1983_HARN_StatePlane_Oregon_North_FIPS_3601


this translates to WKID 2983 HOWEVER because we don’t go in and choose the predefined version of this coordinate system (whose string parses out as : NAD_1983_HARN_StatePlane_Oregon_North_FIPS_3601_Feet_Intl) the service can’t parse the spatial reference into a WKID.

AGS publishing geoprocessing services

  

    

    

      
        
The input must be a feature set as opposed to a feature class.


The tool takes args in the same way traditional geoprocessing scripts do.


Since we’re running the tool in a ‘static’ sense, none of the weighted overlay or workspace params need to be specified, just the input features. Rasters and workspace will be hardcoded.


Server doesn’t lock the script or toolbox — you can edit the script while the service is technically running.  This makes debugging these things less painful.

Tiled vectors

  

    

    

      
        
What is featureserver?


What if a tileserver could select features from a database and create tiled vectors on the fly? 256x256 tiles. Using .NET or something of the ilk…and then adaptively caches those tiles.


It seems that Featureserver serves vectors via REST….not images as previously thought…


This would mean that geometry would not be available but could potentially speed up display.  An open source dynamic layer server.


    

  • Get map size, scale and extent.
    • 

  • Select features from db that intersect extent.
    • 

  • run linear interpolation on each coordinate. (bottleneck)

Needs Assessment :: User Interface to Community Investment Strategy

  

    

    

      
        
Goals: 


    

  1. To be able to run a single zonal statistics task on each of the inputs to the model thus dramatically reducing the time necessary for dynamic geoprocessing on ArcServer.
    


    1.  Can possibly pre-process the inputs but would always have to run zonal stats on the new weighted composite raster.

      

    1. Can possibly set up to multithread locally or distribute task load across multiple SOCs if run on AGS. Constraint: Task must be a discrete model without custom Python coding. Need to somehow accumulate values into the output attribute table.
      2. 

    

    2. 

  2. Provide users with an interface allowing them to use slider bars to adjust the weight of the input layers, submit their formula and visualize the results.
    3. 

  3. Allow users to visualize score breakdowns for each of the input geometries (centers, corridors, custom buffers etc.). Sparklines? Mini bar graphs?
    4. 

  4. Document processes for the construction of each of the input rasters. Provide references to processes if applicable.
    5. 

  5. Express map option geometry choices? Centers, channels, {buffer widths}?
    6.

GTile Creator

  

    

    

      
        
Here is a screenshot of and code for the Google Maps Tile Creator.  I found it much better than ESRI’s crappy caching tools.




The plugin: (Straight from ArcScripts)


GTileCreator


And the script for converting the output of this code to ESRI cache directory format:


On Error Resume Next
    Dim fso, folder, files, NewsFile,sFolder
    Dim StdIn, StdOut
    Set StdIn = WScript.StdIn
    Set StdOut = WScript.StdOut

  Set fso = CreateObject("Scripting.FileSystemObject")
  'Source Folder - where GTileCreator dumped your tiles.
  sFolder = "V:\tempBen"
  'Target Folder - in your arcGIS Server Cache directory.
  tFolder = "S:\arcgisserver\arcgiscache\DG2007\Layers\_alllayers"

  Set folder = fso.GetFolder(sFolder)
  Set files = folder.Files
For each folderIdx In files
     dim fn  : fn=folderIdx.Name
     dim fax : fax = split(folderIdx.Name,"_")
     dim col : col=(fax(1))
     dim row : row=(fax(2))
     dim zoom : zoom=(fax(3))
     dim zoomstr : zoomstr = replace(zoom, ".jpg","")
     dim colstr : colstr=hex(col) & ""
     dim rowstr : rowstr=hex(row) & ""

     if len(zoomstr) <2 then
     while len(zoomstr) <> 2
     zoomstr = "0" & zoomstr
     wend
     end if
     zoomstr = "L" & zoomstr

     while len(colstr) <> 8
     colstr = "0" & colstr
     wend
         colstr = "C" & Lcase(colstr) & ".jpg"

         while len(rowstr) <> 8
         rowstr = "0" & rowstr
         wend
         rowstr = "R" & Lcase(rowstr)

        If Not fso.FolderExists(tFolder & "\" & zoomstr) Then
            fso.CreateFolder (tFolder & "\" & zoomstr)
        End If

        If Not fso.FolderExists(tFolder & "\" & zoomstr & "\" & rowstr) Then
            fso.CreateFolder (tFolder & "\" & zoomstr & "\" & rowstr)
        End If

        'check if the file exists if it does - delete the source
        tFile = tFolder & "\" & zoomstr & "\" & rowstr & "\" & colstr
        If Not fso.FileExists(tfile) Then 'if it doesn't exist
            set fil = fso.getFile(folderIdx)
            if fil.size>1700 then ' and it's greater than 1 kb
                fil.Copy tFile, true ' then copy it over
                StdOut.WriteLine ("copied file: " & fn)
                StdOut.Writeline (vbtab & "to: " & tFile)
            End If
            fil.delete() ' then delete it
            StdOut.Writeline vbtab & "Deleted: " & fn
            set fil = nothing
        Else ' if it does exist
           set fil2 = fso.getFile(tfile) ' get the target file
           set fil = fso.getFile(folderIdx) ' get the source file
            if fil.size > fil2.size then 'if the source is larger than the target
                fil.Copy tFile, true ' so overwrite the target
                StdOut.WriteLine ("copied larger file: " & fn)
            End If
            set fil=nothing
            set fil2=nothing
           fso.deletefile(folderIdx) ' then delete it
           StdOut.Writeline zoomstr & "\" & rowstr & "\" & colstr & " exists"
           StdOut.Writeline vbtab & "Deleted: " & fn
        end If
Next
Set fso = Nothing
