Jul 8 2010

Initial thoughts on tuning ArcGIS Server

Our production GIS web server died again today at 3:50.  I was not surprised.  I had been watching the load not dip below 98% for a sustained period (like an hour).  We rebooted.  Ops had already made their system modifications and I made the following modifications:

Set the maximum number of instances for each service from 2 to 1.

Set all services that are not supposed to be running (File_gdb test services and secured services) to not start automatically and stopped them.

Set the host/SOC capacity to 100 - According to ESRI - “capacity limits the number of service instances that can run on a container machine (SOC)” - I’m not really sure how this translates  - in a way it doesn’t make sense since if you only have 10 services and have set their maximum number of instances to 1 each then it seems to me you would never have more than 10 service instances.?? Am I missing something? I have a hunch that within each ArcSOC instance on the web server there are multiple instances running. This is the only way the capacity number makes sense to me…

If/when the server begins to overheat again we will try the following additional settings:

Setting the services to run in low isolation - according to ESRI - “the isolation level determines whether these instances run in separate processes or share processes” - low isolation implies that service instances will share processes and according to ESRI “can significantly improve memory consumption on your server” but is risky in that “if a process experiences a shutdown or crash” (I don’t know why one would) “all instances sharing the process will be destroyed.”

ALSO:I noticed that the lsass.exe process is coming out as a prime suspect in overheating Mapserver. After googling ArcGIS Server and lsass.exe voila! arrived at this ESRI kb article outlining the correlation, or should I say causation.  It seems that every REST and SOAP request require the use of the LSASS (Local Security Authority Subsystem Service). By creating a new app pool and invoking additional IIS incantations, you can “disable per request authentication.”

Python script to search for string in directory of files

This code searches for a string in a directory and includes subdirectories…Note the “for file in index([yourDirectory])” and “if line.find([yourString]).

import os

def index(directory):
    # like os.listdir, but traverses directory trees
    stack = [directory]
    files = []
    while stack:
        directory = stack.pop()
        for file in os.listdir(directory):
            fullname = os.path.join(directory, file)
            files.append(fullname)
            if os.path.isdir(fullname) and not os.path.islink(fullname):
                stack.append(fullname)
    return files

        continue
    print("."), #just lets us know something is happening
    filecheck = open(file)
    for line in filecheck:
        if line.find(myStr)!= -1:
            print file +" HERE IT IS"
            if not line: break
    filecheck.close()

ArcGIS Server Error Logging

ArcGIS Server Logs are located at “\ArcGIS\Server\User\log”.  A new log file is created each time the SOC is restarted.

Per this blog post, I picked up baretail and the configuration file offered with it.  Makes viewing the log file clearer and also updates the file on the fly.

I’m just starting to correlate the errors and messages with actions performed.  It seems to me that many of them are rather sensationalized.  For instance, if I click on a service that is not started in ArcCatalog the log file logs this:

<Msg time=’2010-07-07T12:01:17’ type=’ERROR’ code=’1013’ target=’ae2009.MapServer’ thread=’420’ elapsed=’0.00000’>Configuration service.server is not started.</Msg>

Thank you ArcGIS Server but I already knew that the service was not started by the little red square on the icon.

Also, when you get messages like this:

<Msg time=’2010-07-07T11:41:28’ type=’WARNING’ code=’2033’ target=’zoning.MapServer’ thread=’3912’>The Server Object instance is shutting down because the Idle Timeout Interval has elapsed</Msg>

It merely means that the service has not been requested again within the idle timeout period - which by default is 30 minutes.  Thanks for the warning that no one used this service for 30 minutes(?) should I care?

At any rate, hopefullly this will provide further ammunition in debugging the spasms that our server has been experiencing lately.

Blocking IP blocks in IIS

Discovered several unauthorized spiders crawling our servers…

I initially setup a detection script to redirect them to a dummy page, but per a co-worker went into IIS and setup deny rules. 

Since the IP addresses were known groups - you can set 75.126.0.0 and match the subnet mask with 255.255.0.0

The IP address groups that were blocked are:

  • 67.228..** 74.86.. 75.126.*. 174.36..** 174.37.. 208.43.*. 69.50.208.*

I don’t know how long this has been going on but I bet it is awhile.  The queries were for complete first/last name pairs.  I ran a few to see if they actually turned up anything but none of them did.

VBA to set all raster symbology in TOC

Takes the symbology of the first raster in the TOC and sets all the rest to it. Handy when you have a ton of sids in there whose noData/background value needs to be set to no color or if you need to set their scale properties the same.

Sub CopyRenderer()

Dim pMxDoc As IMxDocument: Set pMxDoc = Application.Document
Dim pMap As IMap: Set pMap = pMxDoc.FocusMap

Dim pFromLayer As IRasterLayer
Set pFromLayer = pMap.Layer(0)
Dim pToLayer As IRasterLayer
Dim pFromlayerEffects As ILayerEffects
Dim pToLayerEffects As ILayerEffects
Set pFromlayerEffects = pFromLayer
Dim i As Integer

For i = 1 To pMap.LayerCount - 1
    If TypeOf pMap.Layer(i) Is IRasterLayer Then
        Set pToLayer = pMap.Layer(i)
        Set pToLayer.Renderer = pFromLayer.Renderer
        Set pToLayerEffects = pToLayer
        pToLayerEffects.Contrast = pFromlayerEffects.Contrast
        pToLayerEffects.Brightness = pFromlayerEffects.Brightness
        pToLayerEffects.Transparency = pFromlayerEffects.Transparency
        pToLayer.MaximumScale = 0
        pToLayer.MinimumScale = 175000
    End If
Next i

pMxDoc.UpdateContents
pMxDoc.activeView.Refresh

End Sub

Python address validation
