Scientific Notation Regex

While you can’t write a regex to convert scientifically notated numbers to decimal regardless of the exponent, you can write them per number of exponents

For example:

(\d+)\.(\d)(\d+)e\+01
\1\2.\3

(\d+)\.(\d)(\d+)e\+011
\1\2\300

Levenshtein Distance for Excel

Just be sure not to name your module the same as your function. ;)

Public Function Levenshtein(s1 As String, s2 As String)

Dim i As Integer
Dim j As Integer
Dim l1 As Integer
Dim l2 As Integer
Dim d() As Integer
Dim min1 As Integer
Dim min2 As Integer

l1 = Len(s1)
l2 = Len(s2)
ReDim d(l1, l2)
For i = 0 To l1
d(i, 0) = i
Next
For j = 0 To l2
d(0, j) = j
Next
For i = 1 To l1
For j = 1 To l2
If Mid(s1, i, 1) = Mid(s2, j, 1) Then
d(i, j) = d(i - 1, j - 1)
Else
min1 = d(i - 1, j) + 1
min2 = d(i, j - 1) + 1
If min2 < min1 Then
min1 = min2
End If
min2 = d(i - 1, j - 1) + 1
If min2 < min1 Then
min1 = min2
End If
d(i, j) = min1
End If
Next
Next
Levenshtein = d(l1, l2)

End Function

ArcGIS Server Start Stop Batch script

I realize this is not the sexiest blog post but hopefully it will save me a minute or two in the future.

net start ArcServerObjectManager
net start ArcSOCMonitor
Or... 
net stop ArcServerObjectManager
net stop ArcSOCMonitor
For remote services use SC (Service Control) 
SC \\ServerName stop ArcServerObjectManager
SC \\ServerName start ArcServerObjectManager

And the real showstopper:
Start/stop an IIS application pool on a remote box with Python:

from subprocess import call
call(["psexec", r"\\myremotebox",r"C:\Windows\System32\inetsrv\appcmd.exe", "stop", "apppool", "My__Saweet__AppPool"],shell=True)

Helpful GP Environment enumeration

This tip is worth 5hrs of my life…

Always, always, always remember to set the GP workspace and scratch workspace when running a gp process in an SOE.

Handy little snippet to see what your geoprocessing environment values are when running in black box mode.

string envsList = "";
IGpEnumList baz;
baz = gp.ListEnvironments("*");
string envs = baz.Next();
while (envs != "")
{
envsList += envs + "::" + gp.GetEnvironmentValue(envs) + Environment.NewLine;
envs = baz.Next();
}

Failed to Access IIS Metabase

Yellow screen of death =

Description: An unhandled exception occurred during the execution of the current web request. Please review the stack trace for more information about the error and where it originated in the code.

Exception Details: System.Web.Hosting.HostingEnvironmentException: Failed to access IIS metabase.

The process account used to run ASP.NET must have read access to the IIS metabase (e.g. IIS://servername/W3SVC). For information on modifying metabase permissions, please see http://support.microsoft.com/?kbid=267904.

Checked permissions on this dir and ASPNET had read access but this fixed it.

I just ran into this issue when opening manager after installing ArcGIS Server 10 on a new XP box and this fixed my problem.

1. Grant the local ASPNET user rights to C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\Temporary ASP.NET Files

2. At the CMD run this C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\aspnet_regiis.exe -ga ASPNET

3.At the CMD run this iisreset

I’m betting IIS just needed to be reset.
