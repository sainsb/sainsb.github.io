Field Calculator @Area++

This one comes courtesy my co-worker Clint Chiavarini

If you use the Calculate Geometry function in an ArcMap table to calculate area, acres, etc. for shapefiles, here is another way to do it using the field calculator:

  1. At the top of the Field Calculator box, Click “Python” instead of VB Script
  2. Type (without quotes): “!shape.area!” This will give you international square feet (the same as the projection you’re working in).
  3. Or type: !shape.area@acres! to go directly to acres.
  4. The same works for lengths i.e. “!shape.length@kilometers!” or whatnot
    There is no “Calculate Geometry” tool in arctoolbox so if you’re creating a model, you need to use this method in the “Calculate Field” tool.

ArcMap PNG Alpha Channel

I try to use ArcMap as little as possible these days but I just found a new gem at 10 that is something I know people have been asking for for quite some time.

To do this I exported the raster out of Arcmap as a PNG with a highly unlikely transparency color. (Don’t forget to check the box to include the world file)(in this case a .pgw file)

I then opened the image up in Adobe Photoshop and went to town on it, a little feathering here, a little gaussian blur there. Re-open in ArcMap and voila! feathered raster display in ArcMap.

Update : Turns out you’ve been able to do this in ArcMap 9.3.1 as well.

Tell your .NET proxy to accept self-signed certificates

  

    

    

      
        
In attempting to hit our dev and QA ArcGIS Server instances with proxies that manage tokens, we were unable to connect, but when we flipped over to prod, it worked fine. Moreover when we pasted the token request URL that the proxy was requesting into the browser address, voila, we’d get our token.
Come to find out we have a signed certificate on prod but are using self signed SSL certificates on QA and dev.  The proxy code was glancing over the fact that it was receiving both RemoteCertificateChainErrors and RemoteCertificateNameMismatch.  Fiddler was a big help in identifying this…And it didn’t help that we had unconsciously accepted the dev and qa certificates in our trusted list in Firefox.
Adding this code made everything work:


using System;
using System.Collections.Generic;
using System.Text;
using System.Net;
using System.IO;
using System.Net.Security;
using System.Security.Policy;
using System.Security.Cryptography.X509Certificates;
using System.Security.Cryptography;
class Program
{
 static void Main(string[] args)
 {
  Stream s = null;
  StreamReader sr = null;
  HttpWebResponse res = null;
  try{
      //Hook a callback to verify the remote certificate
      ServicePointManager.ServerCertificateValidationCallback =
        new RemoteCertificateValidationCallback(MyCertValidationCb); 

      HttpWebRequest req
        = (HttpWebRequest)
           WebRequest.Create("https://localhost/SecureNoClientCerts/test.htm");

   req.Proxy = null;

   res = req.GetResponse() as HttpWebResponse;
   s = res.GetResponseStream();
   sr = new StreamReader(s, Encoding.UTF8);
   Console.WriteLine(sr.ReadToEnd());
  }
  catch(Exception ex){
   Console.WriteLine(ex);
  }
  finally{
   if(res != null) res.Close();
   if(s != null) s.Close();
   if(sr != null) sr.Close();
  }
 }

  public static bool MyCertValidationCb(
        object sender,
        X509Certificate certificate,
        X509Chain chain,
        SslPolicyErrors sslPolicyErrors)
  {
    if ((sslPolicyErrors & SslPolicyErrors.RemoteCertificateChainErrors)
              == SslPolicyErrors.RemoteCertificateChainErrors)
    {
      return true; //instead of false
    }
    else if ((sslPolicyErrors & SslPolicyErrors.RemoteCertificateNameMismatch)
                    == SslPolicyErrors.RemoteCertificateNameMismatch)
    {
      Zone z;
      z = Zone.CreateFromUrl(((HttpWebRequest)sender).RequestUri.ToString());
      if (z.SecurityZone == System.Security.SecurityZone.Intranet
        || z.SecurityZone == System.Security.SecurityZone.MyComputer)
      {
        return true;
      }
      return false;
    }
    return false;
  } 
}
