Helpful GP Environment enumeration

This tip is worth 5hrs of my life…

Always, always, always remember to set the GP workspace and scratch workspace when running a gp process in an SOE.

Handy little snippet to see what your geoprocessing environment values are when running in black box mode.

string envsList = "";
IGpEnumList baz;
baz = gp.ListEnvironments("*");
string envs = baz.Next();
while (envs != "")
{
envsList += envs + "::" + gp.GetEnvironmentValue(envs) + Environment.NewLine;
envs = baz.Next();
}

Failed to Access IIS Metabase

Yellow screen of death =

Description: An unhandled exception occurred during the execution of the current web request. Please review the stack trace for more information about the error and where it originated in the code.

Exception Details: System.Web.Hosting.HostingEnvironmentException: Failed to access IIS metabase.

The process account used to run ASP.NET must have read access to the IIS metabase (e.g. IIS://servername/W3SVC). For information on modifying metabase permissions, please see http://support.microsoft.com/?kbid=267904.

Checked permissions on this dir and ASPNET had read access but this fixed it.

I just ran into this issue when opening manager after installing ArcGIS Server 10 on a new XP box and this fixed my problem.

1. Grant the local ASPNET user rights to C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\Temporary ASP.NET Files

2. At the CMD run this C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\aspnet_regiis.exe -ga ASPNET

3.At the CMD run this iisreset

I’m betting IIS just needed to be reset.
