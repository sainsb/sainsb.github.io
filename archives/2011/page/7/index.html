URI, URN, URL, URHot

Wikipedia says:

One can classify URIs as locators (URLs), or as names (URNs), or as both. A Uniform Resource Name (URN) functions like a person’s name, while a Uniform Resource Locator (URL) resembles that person’s street address. In other words: the URN defines an item’s identity, while the URL provides a method for finding it. Good enough for me…I think I’ll still just stick with the idea that a URI equals a URL…for all intents and purposes.

Db schema optimization

I recently brought a HUGE table into Arc from Access - we needed the file in DBF format for AccuMail geocoding service. Upon exporting the table from Arc to DBF however, I watched the file grow to over 2gb and then pitifully fail. Why? Because the text fields in the Access table were all 255 chars. How to crank down each of the 20+ fields to it’s longest value length?

SELECT MAX(LEN(site_zip)) as f1_len from table;
Set field lengths accordingly. This would sure be nice though wouldn't it? 
ALTER TABLE t1 Alter Column c1 TEXT(SELECT Int(MAX(LEN(site_zip))) from t1);

Scientific Notation Regex

While you can’t write a regex to convert scientifically notated numbers to decimal regardless of the exponent, you can write them per number of exponents

For example:

(\d+)\.(\d)(\d+)e\+01
\1\2.\3

(\d+)\.(\d)(\d+)e\+011
\1\2\300

Levenshtein Distance for Excel

Just be sure not to name your module the same as your function. ;)

Public Function Levenshtein(s1 As String, s2 As String)

Dim i As Integer
Dim j As Integer
Dim l1 As Integer
Dim l2 As Integer
Dim d() As Integer
Dim min1 As Integer
Dim min2 As Integer

l1 = Len(s1)
l2 = Len(s2)
ReDim d(l1, l2)
For i = 0 To l1
d(i, 0) = i
Next
For j = 0 To l2
d(0, j) = j
Next
For i = 1 To l1
For j = 1 To l2
If Mid(s1, i, 1) = Mid(s2, j, 1) Then
d(i, j) = d(i - 1, j - 1)
Else
min1 = d(i - 1, j) + 1
min2 = d(i, j - 1) + 1
If min2 < min1 Then
min1 = min2
End If
min2 = d(i - 1, j - 1) + 1
If min2 < min1 Then
min1 = min2
End If
d(i, j) = min1
End If
Next
Next
Levenshtein = d(l1, l2)

End Function

ArcGIS Server Start Stop Batch script

I realize this is not the sexiest blog post but hopefully it will save me a minute or two in the future.

net start ArcServerObjectManager
net start ArcSOCMonitor
Or... 
net stop ArcServerObjectManager
net stop ArcSOCMonitor
For remote services use SC (Service Control) 
SC \\ServerName stop ArcServerObjectManager
SC \\ServerName start ArcServerObjectManager

And the real showstopper:
Start/stop an IIS application pool on a remote box with Python:

from subprocess import call
call(["psexec", r"\\myremotebox",r"C:\Windows\System32\inetsrv\appcmd.exe", "stop", "apppool", "My__Saweet__AppPool"],shell=True)

Helpful GP Environment enumeration

This tip is worth 5hrs of my life…

Always, always, always remember to set the GP workspace and scratch workspace when running a gp process in an SOE.

Handy little snippet to see what your geoprocessing environment values are when running in black box mode.

string envsList = "";
IGpEnumList baz;
baz = gp.ListEnvironments("*");
string envs = baz.Next();
while (envs != "")
{
envsList += envs + "::" + gp.GetEnvironmentValue(envs) + Environment.NewLine;
envs = baz.Next();
}
