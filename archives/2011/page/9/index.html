A few notes on Silverlight/WCF/IIS 7

Got my music server running a little better at home and wanted to jot(blot) down some notes on the process. This is total stream of consciousness.

In the dev env. I had added the WCF service from the solution - this created a number of config entries with the http://localhost:[testing server port] - When entering production I had to delete and re-add the service from the production URL to change all the config entries. I’m not sure if that’s a proper deploy tactic but it worked. The next issue was getting the prod url to work remotely - it kept failing at the WCF call….couldn’t figure it out…realized that in the latter step it had used localhost instead of the IP…

RE Sql Server - When you add a database to a project it creates a SQL Server Express mdf file in an App_data directory of your project, when you go to deploy you can publish that mdf to SQL Server and change your connection strings.

It was also necessary for me to change the identity associated with my Application Pool in IIS7 and then Add the NT Authority\NETWORK Service user to the database and give that user read and write access to the db.

Data driven symbol levels

Another benefit of relative elevation values beyond routing is that of symbol levels. If you pop the right fields in your dataset you can do some cool symbology stuff with bridges and overpasses and not have to mess around with manual adjustment (e.g. data driven)

For the purpose of coloring and applying symbolic hierarchies, I setup an extremely simple field calculation to classify street segments by their from/to elevations.

dim lvl
if [F_ELEV]=1 and [T_ELEV]=1 then
lvl=1
elseif ([F_ELEV]=0 and [T_ELEV]=1) OR ([F_ELEV]=1 and [T_ELEV]=0) then
lvl=.5
elseif ([F_ELEV]=2 and [T_ELEV]=1) OR ([F_ELEV]=1 and [T_ELEV]=2) then
lvl=1.5
elseif [F_ELEV]=2 and [T_ELEV]=2 then
lvl=2
elseif ([F_ELEV]=2 and [T_ELEV]=3) OR ([F_ELEV]=3 and [T_ELEV]=2) then
lvl=2.5
elseif [F_ELEV]=3 and [T_ELEV]=3 then
lvl=3
elseif ([F_ELEV]=4 and [T_ELEV]=3) OR ([F_ELEV]=3 and [T_ELEV]=4) then
lvl=3.5
elseif [F_ELEV]=4 and [T_ELEV]=4 then
lvl=4
elseif ([F_ELEV]=4 and [T_ELEV]=5) OR ([F_ELEV]=5 and [T_ELEV]=4) then
lvl=4.5
elseif [F_ELEV]=5 and [T_ELEV]=5 then
lvl=5
End If
