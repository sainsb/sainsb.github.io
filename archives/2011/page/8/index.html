Failed to Access IIS Metabase

Yellow screen of death =

Description: An unhandled exception occurred during the execution of the current web request. Please review the stack trace for more information about the error and where it originated in the code.

Exception Details: System.Web.Hosting.HostingEnvironmentException: Failed to access IIS metabase.

The process account used to run ASP.NET must have read access to the IIS metabase (e.g. IIS://servername/W3SVC). For information on modifying metabase permissions, please see http://support.microsoft.com/?kbid=267904.

Checked permissions on this dir and ASPNET had read access but this fixed it.

I just ran into this issue when opening manager after installing ArcGIS Server 10 on a new XP box and this fixed my problem.

1. Grant the local ASPNET user rights to C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\Temporary ASP.NET Files

2. At the CMD run this C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727\aspnet_regiis.exe -ga ASPNET

3.At the CMD run this iisreset

I’m betting IIS just needed to be reset.

WKT v WKID :: AGS


FeatureLayer fl = new FeatureLayer();
fl.Url = “http://navigator.state.or.us/ArcGIS/rest/services/Framework/Admin_Bounds/MapServer/3“;
int[] foo = {3};
fl.ID = “tempFeatures”;
fl.Visible = true;
fl.InitializationFailed += new EventHandler<EventArgs>(fl_InitializationFailed);
map.Layers.Add(fl);

with:











==Fail.


Why?




Querying features with a WKT as the outSR fails at 9.31 - apparently it’s fixed at 10…


but the large part of the problem revolves around ESRI’s parsing of the projected coordinate system of the data frame when publishing services.


The Oregon Metro Projection is NAD_1983_HARN_StatePlane_Oregon_North_FIPS_3601


this translates to WKID 2983 HOWEVER because we don’t go in and choose the predefined version of this coordinate system (whose string parses out as : NAD_1983_HARN_StatePlane_Oregon_North_FIPS_3601_Feet_Intl) the service can’t parse the spatial reference into a WKID.


SAWEET!


ESRI Forum article relieving much pressure is found here:

C# lambda

  

    

    

      
        
A new layer of the onion peeled off this morning…


The lambda expression, described by Microsoft:


A lambda expression is an anonymous  function that can contain expressions and statements, and can be used to  create delegates or expression tree types.


All lambda expressions use the lambda operator =>,  which is read as “goes to”. The left side of the lambda operator  specifies the input parameters (if any) and the right side holds the  expression or statement block. The lambda expression x => x * x is read “x goes to x times x.”


Discussed at Stack Overflow


Used with the ArcGIS API:


map.Layers.LayersInitialized += (evtsender, args) =>
{
map.ZoomTo(d.InitialExtent);
};
